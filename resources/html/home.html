Configuration

Check out the Configuration section in the documentation, to learn how to configure PSRest. The documentation also covers the best security practices to follow when configuring PSRest to harden your PSRest.

Getting Authenticated

PSRest requires all requests be authorised and authenticated, to learn how to create a Client Id and Secret or generate an Access Token check out the Authentication section in the documentation.

If you've already got an Access Token or Client Id and Secret you can use them to authenticate requests to PSRest. To learn how to do so check out the OAuth Example section in the documentation

Running Commands

All the details regarding how PSRest runs commands and how to format your requests can be found in the Run Commands section in the documentation.

Example using Invoke-WebRequest:

$headers = @{
    "Content-Type" = "application/json";
    "Authorization": "Bearer $accessToken"
}
$body = @{ "command" = "Get-Process" } | ConvertTo-Json
Invoke-WebRequest -Uri http://localhost/run -Method POST -Body $body -ContentType "application/json" -Headers $headers

Example using curl:

curl -X POST http://localhost/run \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer " \
    -d '{"command":"Get-Process"}'

Response:

HTTP: 200
Content-Type : application/json

{
    "data" : [],
    "error" : null
}
